# Архитектура системы

## Архитектурный стиль
В проекте используется микросервисная архитектура.

## Причины выбора
- высокая масштабируемость;
- независимое развитие компонентов;
- отказоустойчивость;
- гибкость внедрения новых функций.

## Общая структура

Система состоит из следующих уровней:

### 1. Уровень представления (Client)
- мобильное приложение (Android / iOS / Flutter);
- отвечает за взаимодействие с пользователем.

### 2. API Gateway
- единая точка входа;
- маршрутизация запросов;
- аутентификация;
- ограничение запросов (rate limiting).

### 3. Backend (микросервисы)

#### Auth Service
- регистрация;
- авторизация;
- JWT;
- восстановление доступа.

#### VPN Core Service
- генерация VPN-конфигураций;
- управление подключениями;
- взаимодействие с Xray-core.

#### Server Management Service
- мониторинг серверов;
- балансировка нагрузки;
- управление VPN-узлами.

#### Billing Service
- тарифы;
- подписки;
- платежи;
- чеки.

#### Monitoring Service
- логирование;
- метрики;
- аудит действий.

### 4. Data Layer
- PostgreSQL — основная БД;
- Redis — кэш и сессии;
- Object Storage — логи и конфигурации.

### 5. VPN-инфраструктура
- Xray-core;
- протокол VLESS;
- TLS-шифрование.

## Поток запроса
1. Пользователь отправляет запрос с клиента.
2. Запрос поступает в API Gateway.
3. Gateway перенаправляет его в нужный сервис.
4. Сервис обрабатывает запрос.
5. Данные сохраняются в БД.
6. Ответ возвращается пользователю.
