# Архитектура системы VPN-Service

## 1. Назначение архитектуры

Архитектура системы определяет структуру программного решения, разделение компонентов, их взаимодействие и распределение ответственности между частями системы.

Основная цель архитектуры:
- обеспечить масштабируемость системы;
- обеспечить отказоустойчивость;
- обеспечить безопасность;
- обеспечить удобство разработки и сопровождения;
- обеспечить возможность расширения функционала.

---

## 2. Архитектурный стиль

Система построена на основе микросервисной архитектуры.

## 2.1 Основные принципы

- каждый сервис отвечает за одну предметную область;
- сервисы взаимодействуют через API;
- сервисы могут разворачиваться независимо;
- данные разделены между сервисами;
- отказ одного сервиса не приводит к полной недоступности системы;
- система может масштабироваться по частям.

---

## 3. Уровень клиента

## 3.1 Назначение

Клиентский уровень представляет собой пользовательский интерфейс системы.

## 3.2 Основные функции

- регистрация пользователя;
- авторизация;
- восстановление доступа;
- просмотр тарифов;
- инициирование оплаты;
- управление подпиской;
- подключение к VPN;
- отключение от VPN;
- просмотр состояния соединения;
- отправка обращений в поддержку;
- получение уведомлений.

## 3.3 Особенности реализации

- клиент не содержит бизнес-логики;
- клиент не взаимодействует напрямую с базой данных;
- клиент работает через API Gateway;
- клиент хранит токен авторизации (JWT);
- клиент отвечает только за отображение данных и ввод информации.

---

## 4. API Gateway

## 4.1 Назначение

API Gateway является единой точкой входа в систему.

## 4.2 Основные функции

- прием всех входящих запросов от клиента;
- маршрутизация запросов к нужным микросервисам;
- проверка токена авторизации;
- базовая аутентификация;
- контроль частоты запросов;
- централизованная обработка ошибок;
- логирование входящих запросов.

## 4.3 Причины использования

- упрощение взаимодействия клиента с системой;
- централизованное управление безопасностью;
- снижение нагрузки на внутренние сервисы;
- возможность добавления дополнительных механизмов защиты.

---

## 5. Backend уровень (микросервисы)

Backend состоит из набора независимых сервисов, каждый из которых реализует определенную бизнес-логику.

---

## 5.1 Auth Service

### Назначение

Сервис отвечает за управление учетными записями пользователей и их аутентификацию.

### Основные функции

- регистрация пользователей;
- проверка уникальности email;
- хеширование паролей;
- авторизация пользователей;
- генерация JWT-токенов;
- проверка токенов;
- восстановление доступа;
- управление сессиями.

### Взаимодействие

- получает запросы от API Gateway;
- взаимодействует с базой данных (таблицы user, role);
- передает токен обратно клиенту через API Gateway.

---

## 5.2 Billing Service

### Назначение

Сервис отвечает за финансовую часть системы.

### Основные функции

- получение списка тарифов;
- управление тарифными планами;
- создание подписок;
- обработка платежей;
- взаимодействие с платежными системами;
- обновление статуса подписки;
- формирование чеков;
- учет баланса пользователя.

### Взаимодействие

- получает запросы через API Gateway;
- взаимодействует с таблицами tariff_plan, subscription, payment, receipt;
- взаимодействует с внешними платежными сервисами.

---

## 5.3 VPN Core Service

### Назначение

Сервис управляет подключением пользователей к VPN.

### Основные функции

- генерация конфигураций подключения;
- выбор сервера;
- установление соединения;
- завершение соединения;
- управление VPN-сессиями;
- взаимодействие с Xray-core;
- проверка статуса подписки перед подключением.

### Взаимодействие

- получает запросы от клиента через API Gateway;
- использует данные о серверах;
- взаимодействует с VPN-инфраструктурой;
- записывает данные о подключениях.

---

## 5.4 Server Management Service

### Назначение

Сервис отвечает за управление VPN-серверами.

### Основные функции

- мониторинг состояния серверов;
- сбор информации о нагрузке;
- распределение нагрузки;
- добавление новых серверов;
- удаление недоступных серверов;
- обновление конфигурации серверов.

### Взаимодействие

- взаимодействует с таблицей server;
- передает данные в VPN Core Service;
- взаимодействует с Monitoring Service.

---

## 5.5 Support Service

### Назначение

Сервис отвечает за обработку обращений пользователей.

### Основные функции

- создание тикетов;
- хранение обращений;
- изменение статусов;
- передача тикетов модераторам;
- обработка ответов;
- уведомление пользователей.

### Взаимодействие

- использует таблицы support_ticket и support_reply;
- взаимодействует с Notification Service;
- получает данные от пользователя через API Gateway.

---

## 5.6 Notification Service

### Назначение

Сервис отвечает за отправку уведомлений пользователям.

### Основные функции

- отправка уведомлений о платежах;
- уведомление об окончании подписки;
- уведомление о ответах поддержки;
- отправка системных сообщений;
- работа с email или push-уведомлениями.

### Взаимодействие

- получает события от других сервисов;
- использует таблицу notification;
- отправляет уведомления пользователю.

---

## 5.7 Monitoring Service

### Назначение

Сервис отвечает за мониторинг и логирование.

### Основные функции

- сбор логов;
- сбор метрик;
- мониторинг серверов;
- отслеживание ошибок;
- аудит действий пользователей и администраторов.

### Взаимодействие

- получает данные от всех сервисов;
- хранит логи в хранилище;
- взаимодействует с системами мониторинга (например Prometheus, Grafana).

---

## 6. Уровень данных (Data Layer)

## 6.1 Назначение

Уровень данных отвечает за хранение и обработку информации.

## 6.2 Основные компоненты

### PostgreSQL

Используется для хранения:
- пользователей;
- ролей;
- тарифов;
- подписок;
- платежей;
- серверов;
- подключений;
- тикетов;
- уведомлений.

### Redis

Используется для:
- хранения сессий;
- кэширования данных;
- ускорения работы системы.

### Object Storage

Используется для:
- хранения логов;
- хранения файлов;
- хранения чеков и документов.

---

## 7. VPN-инфраструктура

## 7.1 Назначение

VPN-инфраструктура обеспечивает защищенное соединение пользователей.

## 7.2 Основные компоненты

- VPN-серверы;
- Xray-core;
- протокол VLESS;
- TLS-шифрование.

## 7.3 Функции

- установление защищенного соединения;
- шифрование трафика;
- распределение нагрузки;
- обеспечение доступа к интернету через VPN.

---

## 8. Взаимодействие компонентов

## 8.1 Общий принцип

1. Пользователь выполняет действие в клиенте.
2. Запрос отправляется в API Gateway.
3. Gateway направляет запрос в нужный сервис.
4. Сервис обрабатывает запрос.
5. Сервис взаимодействует с базой данных.
6. При необходимости сервис взаимодействует с другими сервисами.
7. Ответ возвращается через API Gateway клиенту.

---

## 9. Основные сценарии

## 9.1 Регистрация

- клиент отправляет данные в Auth Service;
- данные сохраняются в базе;
- создается пользователь.

## 9.2 Покупка подписки

- клиент выбирает тариф;
- Billing Service обрабатывает оплату;
- создается подписка;
- отправляется уведомление.

## 9.3 Подключение к VPN

- клиент отправляет запрос;
- проверяется подписка;
- выбирается сервер;
- создается соединение.

## 9.4 Обращение в поддержку

- пользователь создает тикет;
- тикет передается модератору;
- пользователь получает ответ.

---

## 10. Масштабируемость

Система поддерживает:
- горизонтальное масштабирование микросервисов;
- масштабирование VPN-серверов;
- балансировку нагрузки;
- распределение пользователей по серверам.

---

## 11. Безопасность

Основные меры:
- использование JWT;
- хеширование паролей;
- HTTPS/TLS;
- контроль доступа;
- защита от атак.

---

## 12. Отказоустойчивость

Обеспечивается за счет:
- независимости сервисов;
- резервирования серверов;
- мониторинга состояния;
- логирования;
- автоматического масштабирования.
