# Стандарты разработки VPN-Service

## 1. Назначение документа

Документ определяет правила написания кода в проекте VPN-Service.

Цель:
- обеспечить единый стиль кода;
- упростить поддержку;
- повысить читаемость;
- уменьшить количество ошибок;
- ускорить разработку.

---

## 2. Общие принципы

- код должен быть читаемым;
- код должен быть понятным другим разработчикам;
- избегать дублирования;
- соблюдать принципы SOLID;
- писать простой и понятный код;
- избегать излишней сложности.

---

## 3. Именование

## 3.1 Переменные

- используются осмысленные имена;
- стиль: camelCase.

Пример:
userEmail
subscriptionStatus

---

## 3.2 Константы

- используются заглавные буквы;
- слова разделяются через _.

Пример:
MAX_CONNECTIONS
DEFAULT_TIMEOUT

---

## 3.3 Классы

- стиль: PascalCase.

Пример:
UserService
AuthController

---

## 3.4 Методы

- стиль: camelCase;
- имя должно отражать действие.

Пример:
createUser
validateToken
connectToVpn

---

## 4. Структура проекта

Каждый сервис должен иметь структуру:

- controller — обработка HTTP-запросов;
- service — бизнес-логика;
- repository — работа с базой данных;
- model/entity — сущности;
- dto — объекты передачи данных;
- config — конфигурация.

---

## 5. Комментарии

## 5.1 Общие правила

- писать комментарии только там, где это необходимо;
- не дублировать очевидный код;
- описывать сложную логику.

## 5.2 Документация методов

Каждый публичный метод должен иметь описание:

- назначение;
- параметры;
- возвращаемое значение.

---

## 6. Обработка ошибок

- использовать централизованную обработку ошибок;
- не возвращать stack trace пользователю;
- использовать понятные сообщения;
- логировать ошибки.

---

## 7. Работа с API

- использовать REST;
- использовать HTTP методы корректно;
- использовать JSON;
- проверять входные данные;
- возвращать корректные коды ответа.

---

## 8. Работа с базой данных

- использовать ORM;
- не писать SQL без необходимости;
- использовать миграции;
- не дублировать данные;
- использовать индексы.

---

## 9. Безопасность кода

- не хранить секреты в коде;
- использовать переменные окружения;
- валидировать входные данные;
- использовать prepared statements;
- избегать SQL-инъекций.

---

## 10. Логирование

- логировать ошибки;
- не логировать пароли;
- использовать уровни логирования:
  - INFO
  - WARN
  - ERROR

---

## 11. Тестирование

- писать unit-тесты;
- тестировать бизнес-логику;
- использовать mock-объекты;
- покрывать критические части системы.

---

## 12. Работа с Git

- использовать ветки;
- писать понятные commit-сообщения;
- использовать pull request;
- проводить code review.

---

## 13. Code Review

При проверке кода обращать внимание на:
- читаемость;
- архитектуру;
- безопасность;
- производительность;
- наличие тестов.

---

## 14. Форматирование

- использовать единый формат кода;
- соблюдать отступы;
- не писать длинные строки;
- использовать автоматическое форматирование.

---

## 15. Вывод

Соблюдение стандартов разработки позволяет поддерживать высокое качество кода, ускоряет разработку и упрощает сопровождение системы VPN-Service.
