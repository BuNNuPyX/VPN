# VPN-Service — Документация разработчика

## 1. Назначение

Данный раздел содержит документацию разработчика для системы VPN-Service.  
Документация предназначена для понимания архитектуры системы, принципов работы компонентов, структуры проекта и взаимодействия между сервисами.

---

## 2. Описание системы

VPN-Service — это информационная система, предназначенная для предоставления защищенного VPN-доступа пользователям.

Система позволяет:
- регистрировать и авторизовывать пользователей;
- управлять учетными записями;
- просматривать и приобретать тарифные планы;
- выполнять оплату подписки;
- подключаться к VPN-серверам;
- управлять подключениями;
- обращаться в техническую поддержку;
- администрировать систему.

## 4. Архитектурные особенности

Система построена на основе микросервисной архитектуры.  
Каждый сервис отвечает за отдельную функциональную область:

- Auth Service — аутентификация и управление пользователями  
- Billing Service — работа с тарифами и платежами  
- VPN Core Service — управление VPN-подключениями  
- Support Service — обработка обращений пользователей  
- API Gateway — единая точка входа  

Такой подход позволяет:
- масштабировать систему;
- изолировать ошибки;
- упростить сопровождение;
- независимо развивать компоненты.

---

## 5. Основные технологии

В проекте используются следующие технологии:

- Backend: Java (Spring Boot)  
- API: REST (JSON)  
- База данных: PostgreSQL  
- Кэширование: Redis  
- Контейнеризация: Docker  
- Оркестрация: Kubernetes  
- Прокси: Nginx  
- VPN: Xray-core (VLESS)  

---

## 6. Принципы разработки

При разработке системы используются:

- разделение ответственности (SRP);
- слабая связанность компонентов;
- использование DTO для передачи данных;
- централизованная обработка ошибок;
- соблюдение стандартов кодирования;
- логирование и мониторинг.
